## Deployment Steps

### 1. Bootstrap Infrastructure (One-Time)

```bash
# Run bootstrap workflow
gh workflow run "bootstrap-clarity1.yaml" -f environment=dev -f proceed=true

# This will:
# - Create ECR repository: opsera/clarity1
# - Update manifest placeholders with actual values
# - Create ArgoCD repository secret
# - Apply ArgoCD Application for DEV
```

### 2. Deploy Application

```bash
# Option A: Push to main branch (triggers CI/CD automatically)
git add . && git commit -m "feat: Initial deployment" && git push origin main

# Option B: Manually trigger CI/CD
gh workflow run "ci-build-push-clarity1.yaml" -f environment=dev
```

### 3. Setup DNS (One-Time Per Environment)

```bash
# After deployment is successful
gh workflow run "dns-setup-clarity1.yaml" -f environment=dev
```

### 4. Verify Deployment

```bash
# Check application health
curl -I https://opsera-clarity1-dev.agent.opsera.dev

# Run full diagnostics
gh workflow run "diagnostics-clarity1.yaml" -f environment=dev -f diagnostic_type=full-pipeline
```

### 5. Promote to Next Environment

```bash
# Get the image tag from the last successful CI/CD run
# Then promote to QA
gh workflow run "promote-clarity1.yaml" \
  -f target_environment=qa \
  -f image_tag=<IMAGE_TAG> \
  -f confirm_promotion=true

# After QA verification, promote to Staging
gh workflow run "promote-clarity1.yaml" \
  -f target_environment=staging \
  -f image_tag=<IMAGE_TAG> \
  -f confirm_promotion=true
```
